## О проекте

f2p hybrid-casual **match-3** для РуСтор (вариант «соединение цепочки»). Игрок ведёт пальцем по
цепочке соседних одинаковых денежных плиток (≥2) — собирает их в **Баланс**; сверху досыпаются
новые (гравитация). Сверху — Уоррен Баффет «на деньгах» (реакции на комбо). Премиум-валюта 💎.

## Стек

- TypeScript + Vite, **чистый DOM/CSS** (без игрового фреймворка). UI — абсолютно
  позиционированные DOM-вью внутри `#stage` (**384×844**, FIT-масштаб через `transform:scale`).
  Анимации — Web Animations API. Логика — pure-функции `core/*`. Шрифты Nunito/Fredoka
  самохостятся из `public/fonts/` (@font-face в `index.html`).
- `dev`: `npm run dev` → http://localhost:8778 (`strictPort`).
- `build`: `npm run build` (`tsc --noEmit && vite build` → `dist/`). `base:'./'` — относительные пути для WebView/APK.
- `typecheck`: `npm run typecheck`.
- APK — через html2apk (пакует `dist/`).

## Архитектура (кратко)

- **`core/` без DOM** — pure-функции, тестируемые изолированно.
- **`ui/` рисует, `core/` думает** — все мутации SaveData через `core/`. Игровые вью — в `ui/dom/`.
- **`app/GameApp.ts` оркестрирует** — вызывает `core/` и обновляет `ui/`.
- **`config/balance.ts`** — единый источник числовых параметров.
- **Координаты 1-в-1** — DOM-вью позиционируются в дизайн-координатах 384×844.

## Сейвы

- Один ключ `mmatch_save`, поле `schemaVersion`, каскадные миграции с авто-выводом версии.
  Контракт — [docs/SAVES.md](docs/SAVES.md). Любое breaking-изменение формата = новая миграция.
- Балансовые override'ы — отдельный ключ `mmatch_balance_override` (DEV), не в сейве.

## Дев/чит-код — ОБЯЗАТЕЛЬНО гейтить

Механизм (приоритет №1 для Vite-SPA по глобальным правилам): **`import.meta.env.DEV`**.

```ts
if (import.meta.env.DEV) {
  initDevPanel(); // ресурсы, поле, бустеры, override баланса
}
```

При `npm run build` Vite tree-shaking вырезает dev-код — release чистый автоматически.

- **Дев-панель**: кнопка «DEV» (правый верх) + хоткей **~**. Любой новый чит/дев-UI/мок
  оборачивай в `import.meta.env.DEV`. Обычные `console.error` для прод-диагностики — не трогаем.
