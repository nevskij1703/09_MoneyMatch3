# 01 — Кор-петля

## Петля (классический match-3)

1. Поле 5×5 заполнено плитками тиров `1..tierCount` (старт T1..T4), **без готовых матчей** и с
   гарантией хотя бы одного хода.
2. Игрок **свайпает**: зажимает плитку и тянет к ОРТОГОНАЛЬНОМУ соседу (вверх/вниз/влево/вправо)
   — они **меняются местами**.
3. Если обмен создал **линию ≥ `minLine` (=3)** по вертикали/горизонтали ИЛИ **квадрат 2×2**
   одного тира — поле **разрешается каскадами**:
   - совпавшие плитки **схлопываются** → деньги в **Баланс**;
   - **спецматчи рождают спецтайлы** на месте свапа (см. ниже);
   - **гравитация**: уцелевшие плитки (и спецтайлы) падают вниз, сверху **досыпаются новые**
     БЕСПЛАТНО (случайные тиры `1..tierCount`);
   - если падение создало новые матчи — каскад повторяется (комбо растёт).
4. Если обмен **не создал** ни одного матча и ни одна из двух плиток не спецтайл — свап
   **откатывается** (плитки возвращаются назад).
5. Если доступных ходов не осталось — поле **авто-перемешивается** (до появления хода).

Никакой платы за свапы/ходы. Игра активная (нет пассивного дохода/оффлайн-начислений).

## Спецтайлы-бустеры (рождаются из спецматчей)

| Матч | Спецтайл | Эффект при срабатывании |
|---|---|---|
| **Квадрат 2×2** одного тира | 💣 **Бомба** (`bomb`) | сносит область 3×3 вокруг себя (`bombRadius`) |
| **Линия из 5** в ряд (`colorLineLen`) | 🧲 **Магнит** (`color`) | сносит ВСЕ плитки одного тира с поля |

- Линия из 4 — обычный матч (схлопывается, спецтайл НЕ рождается).
- Спецтайл рождается на anchor-клетке (там, где играл игрок — на свапнутой клетке матча).
- **Срабатывает**, когда игрок свайпает спецтайл с соседом (применение на месте; для Магнита
  целевой тир = тир соседа) ИЛИ когда спецтайл попадает под другой схлоп/взрыв (**цепная
  реакция** — задетые спецтайлы срабатывают тоже).

## Комбо (серия схлопов за один ход)

За один ход (свайп) часто схлопывается не одна группа, а серия (каскад досыпки). **Комбо**
считает только **натуральные** матч-схлопы; взрывы спецтайлов (Бомба/Магнит) уровень комбо НЕ
повышают.

- 1-е натуральное исчезновение — без комбо;
- каждое следующее: «**Комбо**» → «**Комбо ×2**» → «**Комбо ×3**» … — крупный баннер над полем
  (`GameApp.showCombo`), затем растворяется;
- денежный бонус шага растёт с уровнем: Комбо **+5%**, ×2 **+6%**, ×3 **+7%**, …
  (`comboBaseBonus` + `comboBonusStep` в [balance.ts](../../src/config/balance.ts)).

## Ценность сбора (деньги)

Формулы — [src/core/economy.ts](../../src/core/economy.ts), параметры —
[src/config/balance.ts](../../src/config/balance.ts) (`match`).

```
ценность плитки t = tierValue(t) × baseTileValue × investmentMultiplier      (tierValue = 2^t)
комбо-множитель    = comboLevel≤0 ? 1 : 1 + comboBaseBonus + (comboLevel−1)·comboBonusStep
ценность шага       = round( Σ ценность схлопнутых плиток × комбо-множитель )
```

- Глубже комбо → выше множитель (Комбо +5%, ×2 +6%, ×3 +7%, …) — поощряет цепные каскады.
- `investmentMultiplier` (старт 1) поднимут будущие **инвестиции** (трата Баланса) — см.
  [08_future.md](08_future.md). Так весь сбор дорожает разом, без инфляции тиров.

При схлопе: поп «+$N» в точке сбора, пульс плашки Баланса, баннер «Комбо ×N» (при комбо) и
реакция Баффета (комбо ≥ 1 или рождение спецтайла).

## Тиры

- Старт — `tierCount = 4` (арт T1..T4, имена/тинты в `balance.namedTiers`).
- Внутренняя стоимость `tierValue(t) = 2^t` (T1=2 … T4=16) — крупнее тир ценнее.
- Расширение набора тиров с прогрессом («новые элементы») — будущее ([08_future.md](08_future.md)).

## Кор-функции

[src/core/match3.ts](../../src/core/match3.ts) (pure, без DOM):
- `areOrthoNeighbors(a, b, cols)` — ортогональная смежность (свап разрешён только между такими).
- `swapCells(field, a, b)` — обмен двух клеток (cells + special).
- `findMatches(field, moved?)` → `{ cleared, spawns }` — линии ≥ `minLine`, квадраты 2×2,
  спавны спецтайлов (≥5 → `color`, 2×2 → `bomb`; anchor — на клетке свапа `moved`).
- `activateSpecial(field, idx, targetTier?)` → клетки, которые сносит спецтайл (bomb 3×3 / color весь тир).
- `expandClearWithSpecials(field, clearedSet)` — цепная реакция спецтайлов в зоне схлопа.
- `applyClear(field, cleared, spawns, tierCount, rng)` → `CascadeStep` — обнуление, установка
  спецтайлов, гравитация + досыпка (дельта для анимации).
- `resolveStep(field, tierCount, rng, moved?)` → `CascadeStep | null` — шаг каскада по матчам поля.
- `wouldSwapMatch(field, a, b)` — даст ли свап матч (спецтайл — всегда; поле не меняется).
- `hasAnyValidMove(field)` / `hasMatchAny(field)` — есть ли ход / есть ли матч (анти-дедлок, settle).
- `makeMatch3Board(cols, rows, tierCount, rng)` — стартовое поле без матчей и с гарантией хода.
