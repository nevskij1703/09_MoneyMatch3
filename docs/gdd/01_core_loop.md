# 01 — Кор-петля

## Петля (классический match-3)

1. Поле **6×5** заполнено плитками тиров `1..tierCount` (старт T1..T4), **без готовых матчей** и
   с гарантией хотя бы одного хода.
2. Игрок **свайпает**: зажимает плитку и тянет к ОРТОГОНАЛЬНОМУ соседу (вверх/вниз/влево/вправо)
   — они **меняются местами**.
3. Если обмен создал **линию ≥ `minLine` (=3)** по вертикали/горизонтали ИЛИ **квадрат 2×2**
   одного тира — поле **разрешается каскадами**:
   - совпавшие плитки **схлопываются** → деньги в **Баланс**; **сложный матч** рождает бустер
     на месте (T/L → 💣, линия-4 → 🚀, квадрат 2×2 → 🛸, линия-5 → 🧲 — см. ниже); **собираемые**
     (💎/⚡/🎁) рядом с этим натуральным схлопом тоже срабатывают (см. «Собираемые объекты»);
   - **гравитация**: уцелевшие объекты падают вниз, сверху **досыпаются новые** БЕСПЛАТНО — обычно
     случайный тир `1..tierCount`, но с шансом (`balance.collect`) — 💎 алмаз / ⚡ молния / 🎁 сейф;
   - если падение создало новые матчи — каскад повторяется (комбо растёт).
4. Если обмен **не создал** ни одного матча — свап **откатывается** (плитки возвращаются назад).
5. Если доступных ходов не осталось — поле **авто-перемешивается** (до появления хода).

Денег за свапы/ходы не берётся. Но **энергия** тратится (−`costPerMove`) в МОМЕНТ свайпа за
успешный ход (свап с матчем или активация бустера); откат энергию не тратит. Игра активная.

## Бустеры

Бустеры существуют в ДВУХ местах:

**1. На поле** — рождаются из СЛОЖНОГО матча (на группу — один бустер по приоритету формы):
| Матч | Бустер | Эффект |
|---|---|---|
| Пересечение линий 3+ (T/L/+) | 💣 **Бомба** (`bomb`) | сносит 3×3 вокруг себя |
| Прямая линия из 4 (`rocketLineLen`) | 🚀 **Ракета** (`rocket-h`/`rocket-v` по ориентации линии) | сносит весь ряд / столбец |
| Квадрат 2×2 | 🛸 **Дрон** (`drone`) | плюс вокруг себя + полёт к приоритетной цели (см. ниже) |
| Линия из 5 (`colorLineLen`) | 🧲 **Магнит** (`magnet`) | собирает весь тир |

Прямая линия 3 без пересечения/квадрата — обычный схлоп без бустера. **Приоритет** (если форма
подходит под несколько): магнит (линия 5) → бомба (T/L) → ракета (линия 4) → дрон (квадрат 2×2).

**Активация бустера на поле:**
- **Свайп** с соседом → бустер сначала ПЕРЕМЕЩАЕТСЯ на клетку соседа (обмен), затем срабатывает
  на НОВОМ месте (матч не нужен). Магнит цель: тир соседней плитки либо случайная ближайшая.
- **Свайп бустера на ПЛИТКУ, образовавший матч** (напр. бомбу свайпнули, и плитка встала в линию-5):
  матч схлопывается И свайпнутый бустер срабатывает **ОДНИМ шагом — бустер активируется СРАЗУ**, без
  задержки на гравитацию (раньше ждал полного оседания — это и был баг с задержкой). Рождённые в этом
  матче бустеры **неуязвимы** к этой активации (выживают и не детонируют — на момент сноса они ещё плитки).
- **Тап** (без свайпа) → срабатывает на СВОЕЙ клетке. Магнит цель: случайная ближайшая плитка.
- **🛸 Дрон**: при ПРЯМОЙ активации (свайп/тап/комбо) на СТАРТЕ взлёта собирает «плюс» (центр + 4 стороны)
  СРАЗУ, затем (подольше) «летит» к цели и активирует её в момент приземления. При ЦЕПНОЙ активации
  (задет волной другого бустера) — плитки «плюса» НЕ сносит, просто спокойно взлетает и летит к цели,
  НО 🎁 **СЕЙФ**, попавший в «плюс» при взлёте, всё равно ВСКРЫВАЕТ (взлёт дрона над сейфом = попадание
  бустера; плитки рядом при этом не трогает). ПРИОРИТЕТ
  цели: сперва 🎁 **СЕЙФ** (дрон охотится на них — сейф открывается только бустером); если сейфов нет —
  обычная плитка-деньги; бустеры/💎/⚡ — лишь фолбэк. Если дронов несколько — **каждый берёт СВОЮ цель**
  (уже занятую другим дроном клетку не выбирает). **Гравитация НЕ стартует, пока дрон не долетел**
  (ждёт ВСЕ полёты, в т.ч. к сейфу) — иначе поле «съедет» под летящим дроном и он попадёт не туда.
- **МГНОВЕННОСТЬ**: бустер срабатывает СРАЗУ (тап / свайп / цепь), БЕЗ «завода»-паузы. У него нет
  пред-анимации — сразу играет своё действие.
- Цепная реакция: бустер, задетый эффектом другого, срабатывает **МГНОВЕННО** в момент, когда эффект
  дошёл до его клетки (`fireTime` = reach-time), и играет СВОЮ полноценную анимацию (бомба на пути
  ракеты взрывается своим 3×3; перпендикулярная ракета летит вдоль своего ряда/столбца; **дрон
  ВЗЛЕТАЕТ** и летит к цели — плитки «плюса» НЕ сносит, но 🎁 сейф в «плюсе» вскрывает). Не
  «всасывается» в волну активатора.

**Анимация активации — у каждого бустера СВОЯ** (см. `planDetonation`/`detonateBlasts` в boardView):
- 💣 **Бомба** — мгновенный снос 3×3 + вспышка-взрыв (`bombBoomMs`).
- 🚀 **Ракета** — два спрайта летят из центра в обе стороны ряда/столбца, гася клетки по мере прохода
  (скорость `rocketFlyMs`).
- 🛸 **Дрон** — убирает «плюс» СРАЗУ (на взлёте — **4 луча света** вдоль убираемых клеток «плюса»),
  затем летит к цели и активирует её в приземлении.
- 🧲 **Магнит** — «выделяет» все клетки тира (сияние-вибрация, `magnetSelectMs`), затем гасит их
  **волной СВЕРХУ ВНИЗ** (шаг между рядами `magnetRowMs`); магнит-комбо так же запускает заспавненные
  бустеры сверху вниз.

**Схлоп обычных матчей** — фишки **резко** сжимаются в ноль и испускают небольшую **радиальную
световую волну** (растёт чуть больше клетки и гаснет). Взрыв бомбы — радиальная волна размером с зону 3×3.

Скорости всех анимаций перемещения — в [config/anim.ts](../../src/config/anim.ts)
(крутятся на лету из дев-панели, вкладка «Анимации», + копия JSON). Падение уцелевших и досыпка
идут с ОДНОЙ скоростью `fallSpeed` и **реакция-каскадом** `reactionMs` (≈100): когда снизу
освобождается слот, предмет над ним стартует через `reactionMs`; стартовав, он сам освобождает место
→ ещё через `reactionMs` стартует следующий (снизу вверх по столбцу). Досыпка сыплется сверху по
одному, а не вся разом. **Каскад без простоя:** следующий матч срабатывает, КАК ТОЛЬКО осели ЕГО
столбцы, а не всё поле — фишки, уже ставшие в ряд, не ждут досыпки в других столбцах (`runNaturalCascade`
подглядывает следующий матч и ждёт только его столбцы).

**Комбо двух бустеров** (ВЫБРАННЫЙ едет на ПРИНИМАЮЩЕГО — тот неподвижен; комбо целиком стартует из
клетки принимающего):
| Свайп | Эффект |
|---|---|
| 💣 + 💣 | взрыв **5×5** вокруг принимающего |
| 💣 + 🚀 | **3 ряда + 3 столбца** вокруг принимающего (вне зависимости от ориентации ракеты) |
| 🚀 + 🚀 | **крест**: 1 ряд + 1 столбец на принимающем |
| 🧲 + 🧲 | собирается **ВСЁ поле** разом |
| 🧲 + любой | собрать случайный тир → на его клетках заспавнить бустер-партнёр → запустить их **волной СВЕРХУ ВНИЗ** |
| 🛸 + 🛸 | взлетают **3 дрона ИЗ клетки принимающего** (каждый к своей цели) |
| 🛸 + 💣/🚀 | дрон взлетает из принимающего и **уносит** бустер, активируя его в точке приземления |

**2. Кнопки-инвентарь** — ряд из 4 кнопок под полем (💣 Bomb / 🛸 Drone / 🚀 Rocket / 🧲 Magnet)
со счётчиком из `SaveData.boosters[id]`. Это **источники бустеров**: зажми кнопку и **перетащи** бустер
на любую клетку поля — «призрак» едет за пальцем, клетка-приёмник подсвечивается рамкой. На отпускании
бустер **СТАВИТСЯ** на клетку: что там стояло (плитка / другой бустер / собираемый / 🎁 сейф) — просто
**ЗАМЕНЯЕТСЯ**, **не собирается и не активируется**. Постановка — **не ход**: энергия не тратится,
каскад не запускается; счётчик кнопки −1. Пустая кнопка (0 шт.) — тряска-отказ. Поставленный бустер
дальше живёт как обычный бустер на поле (активируется свайпом/тапом/цепью). Ракета ставится
горизонтальной. Реальные **эффекты-при-нажатии** (мгновенный blast по полю и т.п.) — не делаем: модель
кнопок — именно постановка на поле.

## Собираемые объекты (💎 алмаз / ⚡ молния / 🎁 сейф)

Досыпаются вместо обычной плитки с шансами `balance.collect` (💎 ~0.5% / ⚡ ~1% / 🎁 ~1%).
💎 алмаз и ⚡ молния **свапаются** как фишки (обмен «прилипает» по матчу, иначе откат; свайп на
**бустер** активирует его и собирает свайпнутый объект). Сами не матчатся и не активируются тапом.
Ловятся в двух случаях: (1) ортогонально рядом схлопнулся **натуральный матч**, ЛИБО (2) **бустер попал
ПРЯМО** по их клетке (пролёт мимо / взрыв рядом, НЕ задевший клетку — не считается).

🎁 **СЕЙФ — НЕПОДВИЖЕН**: его НЕЛЬЗЯ свайпать ни с чем — свайп с участием сейфа просто
игнорируется (сейф даже не дёргается, как стена-препятствие), в матч он не попадает.
Открыть его можно ТОЛЬКО **прямым попаданием бустера** (под ракету / взрыв бомбы / дрон) — соседство с
матчем сейф НЕ открывает. Дрон в ПРИОРИТЕТЕ целится именно в сейфы; кроме того, сейф, попавший в **«плюс»
дрона при взлёте**, тоже вскрывается — даже у ЦЕПНОГО дрона (который обычные плитки «плюса» не сносит). Сейф **ОТКРЫВАЕТСЯ НА СВОЁМ МЕСТЕ**
в момент, когда до него дошёл эффект бустера (награда «вылупляется» тут же); если под ним освободилось —
награда падает уже ПОСЛЕ открытия. Никакого «сперва упасть вниз, потом открыться».

| Объект | При срабатывании |
|---|---|
| 💎 **Алмаз** (`diamond`) | **+1 💎** в баланс алмазов (без комбо/множителей), улетает в карту |
| ⚡ **Молния** (`lightning`) | **+`lightningEnergy` (=3)** энергии (без множителей), улетает в пилюлю Energy |
| 🎁 **Сейф** (`safe`) | ЛУТБОКС: не собирается, а **ОТКРЫВАЕТСЯ** — на месте сейфа появляется награда (бустер / 💎 / ⚡; обычных плиток не бывает). Если под сейфом схлопнулось и он падает по гравитации — он **доезжает до точки приземления и раскрывается ТАМ** (награда баунсит в приземлении, без наложений). Награда **остаётся лежать**, пока не выполнится её условие сбора (возможно в том же ходу) |

Сбор 💎/⚡ — **не умножается** комбо/`investmentMultiplier` (это не «деньги»). Награда из сейфа
сама становится бустером или собираемым и подчиняется их правилам.

## Комбо (серия схлопов за один ход)

**Комбо** = число различных **матч-групп**, исчезнувших за один ход. Считаются и одновременные
(один свайп схлопнул 2 разные группы — это ×2), и каскадные (досыпка создала новый матч).

- уже первая группа = «**Комбо ×1**»; далее «**Комбо ×2**», «**Комбо ×3**» … — крупный баннер
  над полем (`GameApp.updateCombo`), под ним — **накопленная сумма в $** (растёт по ходу);
- бонус **не аддитивный**: берётся % **финального** уровня и применяется ко всей сумме хода
  (×1 → +5%, ×2 → +6%, … итог за ход с ×2 = **+6%**, не 5%+6%); параметры `comboBaseBonus` +
  `comboBonusStep` в [balance.ts](../../src/config/balance.ts);
- в **конце хода** (поле перестало матчиться) накопленные деньги **улетают в карту Баланса**
  (`GameApp.flyMoneyToBalance`), и только тогда Баланс растёт. (Маскот на комбо НЕ реагирует.)

## Ценность сбора (деньги)

Формулы — [src/core/economy.ts](../../src/core/economy.ts), параметры —
[src/config/balance.ts](../../src/config/balance.ts) (`match`).

```
ценность плитки t = tierValue(t) × baseTileValue × investmentMultiplier      (tierValue = t, baseTileValue = 1 → T_t = $t)
baseSum (за ход)   = Σ ценность ВСЕХ схлопнутых за ход плиток
комбо-множитель    = level≤0 ? 1 : 1 + comboBaseBonus + (level−1)·comboBonusStep   (×1 +5%, ×2 +6%, …)
итог хода           = round( baseSum × комбо-множитель(ФИНАЛЬНЫЙ уровень) )         — НЕ аддитивно
```

- Бонус берётся по **финальному** уровню комбо и применяется ко всей `baseSum` хода. Деньги
  зачисляются ОДИН раз в конце хода (`commitMove`).
- `investmentMultiplier` (старт 1, HUD «Income ×N») поднимут будущие **инвестиции/уровень** —
  см. [08_future.md](08_future.md). Так весь сбор дорожает разом, без инфляции тиров.

## HUD и раскладка (макет «Play window», дизайн-холст 390×844)

Сверху вниз: **шапка** (аватар-хомяк + «Hamster Bank» + 🔔/⚙) → **карта** Баланс+Алмазы с
маскотом → **офферы** SALE / Watch Ad → строка **Level / Energy / Income** → **поле 6×5** →
ряд из 4 **кнопок-бустеров** → **нижнее меню** 5 вкладок (**Build** / Tasks / **Play** /
Collections / Shop; стартует на Play). Новые блоки сейчас — визуал + каркас сейва:

- **Level** — `SaveData.level` (старт 1); прокачка — будущее окно.
- **Energy** — `SaveData.energy` / `balance.energy.max`: −`costPerMove` в МОМЕНТ свайпа за успешный
  ход (не по завершении схлопа), реген +`regenAmount` каждые `regenSeconds` (по умолч. **+10 / 10 мин**);
  пополняется ⚡ молнией (+`lightningEnergy`); таймер пилюли = время до следующего +`regenAmount`
  (или «MAX»); при 0 энергии ход заблокирован (см. [core/energy.ts](../../src/core/energy.ts)).
- **Income ×N** — `investmentMultiplier`.
- **Вкладки**: **Build** открывает окно прокачки локации ([02_build.md](02_build.md)), **Play**
  закрывает его и возвращает к игре; **Tasks / Collections / Shop**, а также **Офферы** и **🔔** —
  тап → заглушка (`stubModal`). Жёлтое выделение меню переезжает на нажатую вкладку. Весь UI на
  английском, шрифт **Roboto Flex**.

## Кор-функции

[src/core/match3.ts](../../src/core/match3.ts) (pure, без DOM):
- `areOrthoNeighbors(a, b, cols)` — ортогональная смежность (свап разрешён только между такими).
- `swapCells(field, a, b)` — обмен двух клеток (cells + special).
- `findMatches(field, moved?, rng?)` → `{ cleared, spawns }` — линии ≥ `minLine` и квадраты 2×2;
  на сложную группу 1 бустер (линия-5 → magnet, T/L → bomb, линия-4 → rocket-h/v, 2×2 → drone); anchor — на `moved`.
- `boosterTargets` / `pickNearestTileTier` / `collectBoosterBlasts` — зона поражения и ЦЕПНАЯ
  реакция по-бустерно (в цепь вступают только БУСТЕРЫ; собираемые лишь собираются/открываются).
- `cellsInPlus` / `droneTargets` / `pickDroneFlightTarget` — «плюс» дрона и выбор цели полёта
  (приоритет: 🎁 сейф → обычная плитка-деньги → фолбэк бустеры/💎/⚡).
- `cellsInSquare` / `cellsInRows` / `cellsInCols` / `pickRandomPresentTier` — геометрия зон для
  комбо двух бустеров (5×5 / 3 ряда+3 столбца / крест / магнит-спавн).
- `applyGravityAndRefill` — гравитация + досыпка; новая клетка с шансом — 💎/⚡/🎁 (`balance.collect`).
- `resolveCollectibles` (внутр.) — срабатывание собираемых: 💎/⚡ → сбор (в `CascadeStep.diamonds`/`energy`),
  🎁 → открытие в награду (спавн). Вызывается из `applyClear`. Гард типов — `isBooster`/`isCollectible` (board.ts).
- `countMatchGroups(field, cleared)` — число матч-групп (связные компоненты одного тира) для комбо.
- `applyClear(field, cleared, spawns, tierCount, rng)` → `CascadeStep` — собираемые → обнуление → гравитация + досыпка.
- `resolveStep(field, tierCount, rng)` → `CascadeStep | null` — шаг каскада по матчам поля.
- `wouldSwapMatch(field, a, b)` / `hasMatchAny(field)` / `hasAnyValidMove(field)` — валидация хода, матч, анти-дедлок.
- `makeMatch3Board(cols, rows, tierCount, rng)` — стартовое поле без матчей и с гарантией хода.
