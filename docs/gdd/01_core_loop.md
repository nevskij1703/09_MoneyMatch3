# 01 — Кор-петля

## Петля

1. Поле 6×6 заполнено плитками тиров `1..tierCount` (старт T1..T4), без стартового дедлока.
2. Игрок **зажимает** плитку и **ведёт палец** по СОСЕДНИМ плиткам ТОГО ЖЕ тира — в любом из
   **8 направлений** (орт. + диагонали), путь можно вести **«узелком»** (одну клетку повторно
   выбрать нельзя) — строится цепочка (подсветка клеток + золотая соединительная линия). Ведя на
   предпоследнюю клетку, можно **откатить** последний шаг.
3. На **отпускании**, если длина цепочки ≥ `minChain` (=2):
   - плитки цепочки **собираются** → деньги идут в **Баланс**;
   - **гравитация**: уцелевшие плитки столбца падают вниз, сверху **досыпаются новые**
     БЕСПЛАТНО (случайные тиры `1..tierCount`).
4. Если хода не осталось (нет ни одной пары соседних одинаковых) — поле авто-перемешивается.

Никакой платы за спавн/ходы. Игра активная (нет пассивного дохода/оффлайн-начислений).

## Ценность сбора (деньги)

Формулы — [src/core/economy.ts](../../src/core/economy.ts), параметры —
[src/config/balance.ts](../../src/config/balance.ts) (`match`).

```
ценность плитки t = tierValue(t) × baseTileValue × investmentMultiplier      (tierValue = 2^t)
комбо-множитель    = 1 + max(0, длина − minChain) × comboStep
ценность цепочки   = round( Σ ценность плиток × комбо-множитель )
```

- Длиннее цепочка → выше комбо-множитель (поощряет крупные сборы).
- `investmentMultiplier` (старт 1) поднимут будущие **инвестиции** (трата Баланса) — см.
  [08_future.md](08_future.md). Так весь сбор дорожает разом, без инфляции тиров.

При сборе: поп «+$N» в точке сбора, пульс плашки Баланса, реакция Баффета на комбо (длина ≥ 4).

## Тиры

- Старт — `tierCount = 4` (арт T1..T4, имена/тинты в `balance.namedTiers`).
- Внутренняя стоимость `tierValue(t) = 2^t` (T1=2 … T4=16) — крупнее тир ценнее.
- Расширение набора тиров с прогрессом («новые элементы») — будущее ([08_future.md](08_future.md)).

## Кор-функции

[src/core/match3.ts](../../src/core/match3.ts):
- `neighbors(idx, field, diagonal)` — смежные клетки (при `diagonal=true` — 8 соседей: орт.+диагонали).
- `canExtendChain(chain, idx, field, diagonal)` — кандидат в цепочку (тот же тир, смежен с хвостом, ещё не в цепочке).
- `collectChain(field, chain)` → собранные тиры, обнуляет клетки.
- `applyGravityAndRefill(field, tierCount, rng)` → `{ falls, spawns }` для анимаций.
- `hasAnyChain(field, minChain, diagonal)` — есть ли ход (для авто-перемешивания).
- `makeMatch3Board(cols, rows, tierCount, rng)` — стартовое поле с гарантией хода.
