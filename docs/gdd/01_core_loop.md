# 01 — Кор-петля

## Петля (классический match-3)

1. Поле **6×5** заполнено плитками тиров `1..tierCount` (старт T1..T4), **без готовых матчей** и
   с гарантией хотя бы одного хода.
2. Игрок **свайпает**: зажимает плитку и тянет к ОРТОГОНАЛЬНОМУ соседу (вверх/вниз/влево/вправо)
   — они **меняются местами**.
3. Если обмен создал **линию ≥ `minLine` (=3)** по вертикали/горизонтали ИЛИ **квадрат 2×2**
   одного тира — поле **разрешается каскадами**:
   - совпавшие плитки **схлопываются** → деньги в **Баланс**; **сложный матч** рождает бустер
     на месте (T/L → 💣, квадрат 2×2 → 🚀, линия-5 → 🧲 — см. ниже);
   - **гравитация**: уцелевшие плитки (и бустеры) падают вниз, сверху **досыпаются новые** БЕСПЛАТНО
     (случайные тиры `1..tierCount`);
   - если падение создало новые матчи — каскад повторяется (комбо растёт).
4. Если обмен **не создал** ни одного матча — свап **откатывается** (плитки возвращаются назад).
5. Если доступных ходов не осталось — поле **авто-перемешивается** (до появления хода).

Никакой платы за свапы/ходы. Игра активная (нет пассивного дохода/оффлайн-начислений).

## Бустеры

Бустеры существуют в ДВУХ местах:

**1. На поле** — рождаются из СЛОЖНОГО матча (на группу — один бустер по приоритету формы):
| Матч | Бустер | Эффект |
|---|---|---|
| Пересечение линий 3+ (T/L/+) | 💣 **Бомба** (`bomb`) | сносит 3×3 вокруг себя |
| Квадрат 2×2 | 🚀 **Ракета** (`rocket-h`/`rocket-v`, рандом) | сносит весь ряд / столбец |
| Линия из 5 (`colorLineLen`) | 🧲 **Магнит** (`magnet`) | собирает весь тир |

Прямая линия 3/4 без пересечения/квадрата — обычный схлоп без бустера. Бустер на поле
**срабатывает** свайпом с любым соседом (матч не нужен; свайп двух бустеров активирует оба) или
цепной реакцией (взрыв/ракета/магнит). Магнит цель: тир соседа по свайпу либо ближайшая плитка.

**2. Кнопки-инвентарь** — ряд из 4 кнопок под полем (💣 Bomb / 🛸 Drone / 🚀 Rocket / 🧲 Magnet)
со счётчиком из `SaveData.boosters[id]`. Сейчас тап → **заглушка** (`GameApp.onBooster`); реальные
эффекты кнопок — **будущая фаза**.

## Комбо (серия схлопов за один ход)

**Комбо** = число различных **матч-групп**, исчезнувших за один ход. Считаются и одновременные
(один свайп схлопнул 2 разные группы — это ×2), и каскадные (досыпка создала новый матч).

- уже первая группа = «**Комбо ×1**»; далее «**Комбо ×2**», «**Комбо ×3**» … — крупный баннер
  над полем (`GameApp.updateCombo`), под ним — **накопленная сумма в $** (растёт по ходу);
- бонус **не аддитивный**: берётся % **финального** уровня и применяется ко всей сумме хода
  (×1 → +5%, ×2 → +6%, … итог за ход с ×2 = **+6%**, не 5%+6%); параметры `comboBaseBonus` +
  `comboBonusStep` в [balance.ts](../../src/config/balance.ts);
- в **конце хода** (поле перестало матчиться) накопленные деньги **улетают в карту Баланса**
  (`GameApp.flyMoneyToBalance`), и только тогда Баланс растёт; маскот реагирует (комбо ≥ 2).

## Ценность сбора (деньги)

Формулы — [src/core/economy.ts](../../src/core/economy.ts), параметры —
[src/config/balance.ts](../../src/config/balance.ts) (`match`).

```
ценность плитки t = tierValue(t) × baseTileValue × investmentMultiplier      (tierValue = 2^t)
baseSum (за ход)   = Σ ценность ВСЕХ схлопнутых за ход плиток
комбо-множитель    = level≤0 ? 1 : 1 + comboBaseBonus + (level−1)·comboBonusStep   (×1 +5%, ×2 +6%, …)
итог хода           = round( baseSum × комбо-множитель(ФИНАЛЬНЫЙ уровень) )         — НЕ аддитивно
```

- Бонус берётся по **финальному** уровню комбо и применяется ко всей `baseSum` хода. Деньги
  зачисляются ОДИН раз в конце хода (`commitMove`).
- `investmentMultiplier` (старт 1, HUD «Income ×N») поднимут будущие **инвестиции/уровень** —
  см. [08_future.md](08_future.md). Так весь сбор дорожает разом, без инфляции тиров.

## HUD и раскладка (макет «Play window», дизайн-холст 390×844)

Сверху вниз: **шапка** (аватар-хомяк + «Hamster Bank» + 🔔/⚙) → **карта** Баланс+Алмазы с
маскотом → **офферы** SALE / Watch Ad → строка **Level / Energy / Income** → **поле 6×5** →
ряд из 4 **кнопок-бустеров** → **нижнее меню** 5 вкладок (Постройка / Задачи / **Игра** /
Коллекции / Магазин; центр «Игра» выделен). Новые блоки сейчас — визуал + каркас сейва:

- **Level** — `SaveData.level` (старт 1); прокачка — будущее окно.
- **Energy** — `SaveData.energy` / `balance.energy.max`: −`costPerMove` за успешный ход, реген
  +`regenAmount` каждые `regenSeconds` (по умолч. **+10 / 10 мин**); таймер пилюли = время до
  следующего +`regenAmount` (или «MAX»); при 0 энергии ход заблокирован (см. [core/energy.ts](../../src/core/energy.ts)).
- **Income ×N** — `investmentMultiplier`.
- **Офферы**, **🔔**, **вкладки** — тап → заглушка (`stubModal`); жёлтое выделение меню переезжает
  на нажатую вкладку. Весь UI на английском, шрифт **Roboto Flex**.

## Тиры

- Старт — `tierCount = 4` (арт T1..T4, имена/тинты в `balance.namedTiers`).
- Внутренняя стоимость `tierValue(t) = 2^t` (T1=2 … T4=16) — крупнее тир ценнее.
- Расширение набора тиров с прогрессом — будущее ([08_future.md](08_future.md)).

## Кор-функции

[src/core/match3.ts](../../src/core/match3.ts) (pure, без DOM):
- `areOrthoNeighbors(a, b, cols)` — ортогональная смежность (свап разрешён только между такими).
- `swapCells(field, a, b)` — обмен двух клеток (cells + special).
- `findMatches(field, moved?, rng?)` → `{ cleared, spawns }` — линии ≥ `minLine` и квадраты 2×2;
  на сложную группу 1 бустер (линия-5 → magnet, T/L → bomb, 2×2 → rocket-h/v); anchor — на `moved`.
- `boosterTargets` / `pickNearestTileTier` / `expandClearWithSpecials` — зона поражения и цепная
  реакция бустеров (активация свайпом/цепью).
- `countMatchGroups(field, cleared)` — число матч-групп (связные компоненты одного тира) для комбо.
- `applyClear(field, cleared, spawns, tierCount, rng)` → `CascadeStep` — обнуление, гравитация + досыпка.
- `resolveStep(field, tierCount, rng)` → `CascadeStep | null` — шаг каскада по матчам поля.
- `wouldSwapMatch(field, a, b)` / `hasMatchAny(field)` / `hasAnyValidMove(field)` — валидация хода, матч, анти-дедлок.
- `makeMatch3Board(cols, rows, tierCount, rng)` — стартовое поле без матчей и с гарантией хода.
- `boosterTargets` / `pickNearestTileTier` / `expandClearWithSpecials` — заготовки эффектов бустеров (для будущей логики кнопок; полем не используются).
