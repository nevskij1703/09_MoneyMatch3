# 02 — Build (прокачка локации)

Окно **«Build»** (вкладка слева в нижнем меню) — мета-петля «играй → зарабатывай → улучшай».
Открывается тапом по вкладке **Build** (полноэкранный оверлей поверх экрана Play, лежит ПОД нижним
меню), закрывается вкладкой **Play**. Реализация: [buildWindowView.ts](../../src/ui/dom/buildWindowView.ts)
(вью), [core/build.ts](../../src/core/build.ts) (логика, PURE), [balance.ts](../../src/config/balance.ts)
(секция `build`); сейв — `SaveData.build.steps` (см. [SAVES.md](../SAVES.md)).

## Раскладка (дизайн-холст 390×844, макет Figma «Build window»)

Сверху вниз: **счётчики** (аватар + money / 💎 / energy с кнопками «+» + ⚙) → **ряд локаций**
(5 точек + прогресс-бар) → **арт активной локации** (фон) → **карточки построек** (5 шт.) → общее
**нижнее меню** 5 вкладок (то же, что на Play; жёлтое выделение — на Build). Счётчики и кнопки «+»,
⚙, закрытые локации — тап → заглушка (`onStub`).

## Локации (ряд сверху)

5 точек: **Office** (пройдена ✓), **Jet** (текущая, подсвечена жёлтым кольцом), **Garage /
Penthouse / Yacht** (закрыты 🔒). Пока **активна хардкодно одна** локация
(`balance.build.activeLocation` = Jet); остальные — визуал (тап по закрытой → заглушка).
Прогресс-бар = Σ шагов построек / максимум (`locationProgress`). Будущее: разблокировка локаций, у
каждой свой арт-фон и свой набор построек.

## Постройки (карточки)

Активная локация имеет 5 построек (`balance.build.buildings`, порядок = по возрастанию цены):
**Seats → Table → Shelf → TV → Dishes**. Каждую можно прокачать **до 5 раз**
(`upgradesPerBuilding`). Карточка: имя, арт, индикатор шагов (n из 5 точек), цена следующего шага.

- **Доступна** (зелёная рамка + свечение): не максимум И хватает Баланса → тап = **прокачать**
  (списать цену, шаг +1, сохранить, обновить карточки).
- **Недоступна** (синяя, текст приглушён): не хватает Баланса → тап = тряска карточки + пульс
  счётчика денег.
- **MAX** (5/5): прокачана полностью → тап = лёгкий баунс (no-op).

Прокачка тратит **Баланс** (главную валюту match-3); карта баланса на экране Play синхронизируется
(`onChange`). Влияние прокачки на доход — **будущее** (пока это чистый money-sink + прогресс-арт).

## Цены (геометрическая лесенка)

Цена шага — `buildUpgradeCost(b, step)` ([core/build.ts](../../src/core/build.ts)): **равномерная
лесенка в лог-масштабе** по всем `building × step` ячейкам, от `costMin` (1-й шаг 1-й постройки) до
`costMax` (последний шаг последней). По умолчанию `costMin = 1K`, `costMax = 9.99Qa` — каждый
следующий шаг ≈ ×3.5. Цена округляется до 3 значащих цифр (списанная сумма ТОЧНО == показанная
`formatMoney`). Ребаланс — менять `costMin`/`costMax` (или заменить формулу) в `balance.build`;
**цены ещё будут меняться, и не раз**.

## Текущее состояние / будущее

- **Сейчас**: рабочая прокачка (трата Баланса, шаги 0..5, сохранение, все состояния карточек),
  1 активная локация, прогресс-бар по шагам.
- **Будущее**: влияние построек/локаций на доход (`investmentMultiplier`), несколько локаций с
  разблокировкой, награды за полную прокачку локации, анимации стройки, альтернативные цены (💎).
